# feat: 解析与映射链路改造为前端直连 OpenAI（fix_res）

**公用约定**：以 `.cursor/prd/agent-dev-guide-summary.md` 为准。仓库级 Agent 基线（Cursor 始终应用）：`.cursor/rules/creator-cfo-always.mdc`。

## 平台描述

当前链路为：前端 -> Vercel 接口 -> OpenAI。  
本需求改为：前端 -> OpenAI（直接调用）。

目标：

- 去除 Vercel 中转依赖；
- 保持现有业务流程与字段映射结果一致；
- 最小改动完成链路替换与回归。

> 说明：本仓库当前阶段无独立后端。本 feat 仅调整请求链路，不引入新服务。

## 存储与契约影响（必填）

- **结构化 DB / 文件仓**：`无变更`
- **须同步的工件**：
  - [ ] `packages/storage/src/contracts.ts`
  - [ ] `docs/contracts/`
  - [ ] `packages/schemas`
  - [x] 自动化测试（至少 1 条主路径 + 1 条错误路径）
- **说明**：不改本地存储契约；如 OpenAI 返回结构发生变化，需补充 DTO 约束并更新测试。

## 需求描述

### 目标

1. 前端不再调用 Vercel 解析/映射接口，改为直接请求 OpenAI。
2. 业务流程不变：上传/解析/映射/回填顺序不变。
3. UI 行为不变：成功态、失败态、加载态语义保持一致。

### OpenAI 调用约束

- 模型、endpoint、headers 统一收敛在单一请求模块（禁止多处散落实现）。
- 请求与响应统一 JSON 格式，保持与现有页面消费结构兼容。
- 错误统一分层：网络错误 / OpenAI 错误 / 解析失败，供 UI 明确提示。

### 安全与环境约束（必须写清）

- 本需求按“前端直连 OpenAI”实施，默认会暴露客户端可见凭证。
- 因此本改造仅用于当前阶段（开发/内测）链路简化；生产化方案需在后续 PRD 明确（例如服务端注入或临时凭证）。
- 禁止在代码仓库提交明文密钥。

### 配置约定（建议）

- `EXPO_PUBLIC_OPENAI_BASE_URL`（默认 `https://api.openai.com/v1`）
- `EXPO_PUBLIC_OPENAI_MODEL`
- `EXPO_PUBLIC_OPENAI_API_KEY`（仅开发/内测使用，禁止提交仓库）

## 范围边界

**包含**：

- 前端请求层从 Vercel 改为 OpenAI 直连；
- 环境变量接入；
- 主链路与错误链路回归。

**不包含**：

- 新后端服务建设；
- 数据库/文件仓契约改造；
- 解析能力重构（prompt 大改、模型策略重设计）；
- 非本需求相关 UI 重构。

## 验收标准

- [ ] 前端已移除对 Vercel 解析/映射接口的主路径依赖。
- [ ] 前端可直接调用 OpenAI 完成解析与映射，并返回现有页面可消费的数据结构。
- [ ] 改造后主流程结果与改造前一致（字段回填与状态流转一致）。
- [ ] 网络异常、429、5xx、响应格式异常均有明确错误提示，无静默失败。
- [ ] 配置走环境变量，仓库无明文密钥。
- [ ] 至少 1 条主路径测试 + 1 条错误路径测试通过。
- [ ] `pnpm lint`、`pnpm typecheck`、`pnpm test` 通过。

## 实施步骤（给 Agent 的最小执行指引）

1. 定位当前 Vercel 调用入口与数据适配层。
2. 抽离/统一 OpenAI 客户端调用模块（请求参数、鉴权、错误归一）。
3. 原位替换现有调用，保持输入输出结构不变。
4. 回归主路径（上传 -> 解析 -> 映射 -> 回填）与关键失败路径。
5. 更新自动化测试并提交证据。
