# 外部参考：会计与记账开源项目（非真源）

本文档用于说明仓库在编写 PRD、优化需求描述、补充产品流或校正计算口径时，可以如何有限参考外部开源项目。

这些项目仅用于启发式参考，不是本仓库的 Source of Truth。

## 适用边界

可参考：

- 产品交互思路
- 模块拆分方式
- 记账与会计计算口径
- 异常场景与边界条件
- 中小企业记账产品的信息架构

不可直接照搬：

- 数据库结构
- 包结构与代码组织
- 后端架构
- 框架实现
- 与本仓库阶段假设冲突的能力设计

本仓库真源仍然是：

- `docs/contracts/`
- `packages/storage/src/contracts.ts`
- `packages/schemas/src/index.ts`

若外部参考与当前 PRD、当前契约或当前阶段假设冲突，以本仓库文档与契约为准。

## 项目 1：GnuCash

- 仓库：<https://github.com/gnucash/gnucash>
- 项目类型：个人与小企业的复式记账软件
- 更适合参考的方向：
  - 复式记账
  - 金额汇总
  - 余额计算
  - 分录平衡
  - 会计口径一致性
  - 借贷方向、累计值、账户视角的异常场景

推荐使用方式：

- 当 PRD 涉及金额计算、汇总修复、余额一致性、借贷方向、会计规则时，优先参考 `GnuCash`
- 在 PRD 中应把引用落到“计算口径”或“异常处理”层面，而不是要求“实现方式与 GnuCash 一致”

## 项目 2：Akaunting

- 仓库：<https://github.com/akaunting/akaunting>
- 项目类型：面向中小企业和自由职业者的在线会计软件
- 更适合参考的方向：
  - 中小企业记账产品的信息架构
  - 表单流程
  - 账单、费用、报表等模块拆分
  - 页面组织与产品流
  - 常见会计产品的交互结构

推荐使用方式：

- 当 PRD 涉及页面流、模块组织、账单/费用/报表体验、信息架构时，优先参考 `Akaunting`
- 在 PRD 中应把引用落到“产品流与模块结构”层面，而不是要求“界面长得像 Akaunting”

## 在 PRD 中的推荐写法

建议在 feat PRD 中新增如下小节：

```md
## 外部参考（非真源）

以下项目仅用于产品交互、业务流程、计算口径与异常处理的启发式参考，不作为本仓库契约、数据结构或实现真源：

- GnuCash: https://github.com/gnucash/gnucash
  - 重点参考：复式记账、金额汇总、余额计算、会计口径一致性
- Akaunting: https://github.com/akaunting/akaunting
  - 重点参考：中小企业记账产品的信息架构、表单流程、报表与业务模块组织

使用边界：
- 不直接迁移其数据库结构、包结构、后端实现
- 本仓库仍以 `docs/contracts/`、`packages/storage/src/contracts.ts`、`packages/schemas/src/index.ts` 为真源
- 若外部参考与当前 PRD 冲突，以当前 PRD 与本仓库契约为准
```

## 引用建议

- 修复计算、余额、借贷方向、汇总口径：优先点名 `GnuCash`
- 优化账单、费用、报表页面流或模块组织：优先点名 `Akaunting`

推荐补充句式：

- “本需求在计算口径与异常场景上参考 GnuCash，但不引入其数据模型或实现结构。”
- “本需求在产品流与模块组织上参考 Akaunting，但仍按本仓库本地优先架构落地。”
