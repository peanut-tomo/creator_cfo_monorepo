# Context Snapshot

## Version

- version: 0.4.0
- updated_at: 2026-04-01
- scope: deliver `feat_upload` with local-first upload intake, iOS native OCR, parse review persistence, and SQLite-backed Home data

## Active Decisions

- Monorepo remains the baseline.
- Frontend stays on Expo + React Native + Expo Router.
- There is still no backend in the current phase.
- Storage remains schema-first: local SQLite plus the on-device file vault.
- iOS uses a local Expo Module backed by Apple Vision OCR in development builds.
- Android and Web use fallback parsing instead of native OCR.
- `entity-main` is the default local entity for upload, parse, and Home aggregation.
- PDF stays on fallback parsing in this slice.

## Implemented Structure

- `modules/ios-ocr`: local Expo Module for iOS image OCR with Apple Vision.
- `packages/storage/src/contracts.ts`: `evidences` contract extended with `file_path`, `parse_status`, and `extracted_data`, plus upload-path helper and parse-queue index.
- `packages/schemas/src/index.ts`: parse-status, parser-kind, and extracted-data DTOs.
- `apps/mobile/src/features/ledger/*`: local upload intake, parse queue, retry, confirmation flow, and platform-specific runtimes.
- `apps/mobile/src/features/home/*`: Home aggregation from persisted records with refresh and pagination.
- `tests/smoke/README.md`: updated iOS OCR and Android/Web fallback smoke paths.
