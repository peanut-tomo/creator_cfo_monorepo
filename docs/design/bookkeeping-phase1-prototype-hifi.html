CreatorCFO · Phase 1 Hi-Fi

只补记账、上传、解析，不改原有底部菜单

这版高保真原型严格贴合当前仓库：保持 Home / Ledger / Discover / Profile 四个底部菜单不变，只在对应页面里补充记账、上传、解析能力。整体视觉延续项目现有暖白底、墨蓝字、低饱和绿色强调色的本地优先产品气质。

本期边界

只做收入、费用、发票记录；只做文件上传与解析确认；只做本地工作流。不引入税务、自动同步、银行直连和新的底部一级导航。

Design Focus 更像真实 iPhone App，而不是概念板

我把页面密度、卡片层级、状态标签、底部导航、记录列表和上传流程都做得更接近真实移动产品，以便你后续继续做视觉讨论或落到 React Native 页面实现。

对应关系

Home 增强概览和待解析提醒；Ledger 承接主记账和上传入口；Discover 承接上传历史、解析模板和工具入口；Profile 保持原样，只作信息一致性展示。

UX Notes

这版 UI 的关键调整

  • 把首页从说明页变成财务状态和待处理动作入口。
  • 把 Ledger 明确成主工作区，承接新增记录和上传。
  • 把 Parse Review 放到上传之后，形成真实工作流。
  • 把 Discover 控制在工具入口页，不扩成新业务线。

Phase 1 Scope

当前只做的能力

  • 收入、费用、发票记录。
  • 上传票据、statement、invoice file。
  • 解析字段并人工确认。
  • 记录和附件的关联与状态管理。
